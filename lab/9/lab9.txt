Программирование графического интерфейса в Windows.

Графический интерфейс является неотъемлемой частью Windows. WinAPI предоставляет средства для программирования графического интерфейса в пользовательских приложениях.
Основные функции предоставляются библиотеками:
user32.dll
gdi32.dll
Графический интерфейс реализован на уровне подсистемы окружения Windows. Основная функциональность реализована в компоненте этой системы уровня ядра (win32k.sys), а библиотеки пользовательского режима предоставляют интерфейс для этого компонента.

Разделы MSDN:
Win32 и разработка COM -> Graphics and Multimedia -> GDI -> Windows GDI
Win32 и разработка COM -> Graphics and Multimedia -> GDI+

Основным элементом графического интерфейса является окно. Все, что мы видим на экране, является окнами. Любые действия с окном приводят к посылке сообщения окну. Обработку оконных сообщений осуществляет оконная процедура, определённая создалем окна. Программирование графического интерфейса по сути заключается в создании необходимых окон и обработки оконных сообщений, им посылаемых.

Окна являются главными элементами графической подсистемы, но помимо них существуют другие типы объектов:
- картинки (bitmap)
- иконки (icon)
- кисти (brush)
- перья (pen)
- шрифты (font)
- палитры (logical Palette)
- регионы (region)

Перед созданием окна, нужно зарегистрировать его класс, указав некоторые свойства, общие для некоторого семейства окон. Система создаёт некоторые стандартные классы окон (кнопки, текстовые поля, надписи и др.), которые можно использовать в качестве готовых компонентов интрефейса. Они называются управляющими элементами или контролами (controls).
Описание в MSDN:
Win32 и разработка COM -> Development Guides -> Windows API -> Overview of the Windows API -> Windows User Interface
